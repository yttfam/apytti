POST /api/ask

Send a prompt to the active backend (or override per request). Pass session_id from a previous response to continue a conversation.

Request

FieldTypeDescription
promptstringrequiredThe prompt to send
backendstringoptionalOne of: claude, copilot, gemini, ollama. Defaults to active backend.
session_idstringoptionalSession ID to resume
modelstringoptionalModel override
effortstringoptionalEffort level (Claude/Copilot only)
{
  "prompt": "translate hello world to french",
  "backend": "ollama",
  "model": "llama3.2"
}

Response

FieldTypeDescription
responsestringThe model's response
session_idstringSession ID (pass back to continue)
cost_usdnumber?Estimated cost (null when not applicable, e.g. subscription/local)
backendstringBackend that handled the request
errorstring?Error message if something went wrong
{
  "response": "\"Hello world\" in French is \"Bonjour le monde\".",
  "session_id": "7bb1d5fb-1fe1-41d8-abf7-1d0e730eefb6",
  "cost_usd": null,
  "backend": "ollama",
  "error": null
}

Configuration

Backends are configured in ~/.apytti/config.toml. Run apytti setup for an interactive menu.

active = "claude"

[backends.claude]
enabled = true
model = "sonnet"
effort = "low"
skip_permissions = true

[backends.ollama]
enabled = true
endpoint = "http://localhost:11434"
model = "llama3.2"

Session flow

  1. POST without session_id → backend creates one
  2. Response includes session_id
  3. Pass it back in next request to continue

Sessions for Claude/Copilot/Gemini are managed by their respective CLIs. Ollama sessions are kept in apytti's memory (lost on restart).
